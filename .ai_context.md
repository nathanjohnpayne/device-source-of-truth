## Do Not Touch

| File/Path | Reason |
|-----------|--------|
| `dist/` | Vite build output — never edit manually; run `npm run build` to regenerate |
| `packages/contracts/src/index.ts` | The canonical type source — changes here propagate everywhere; requires careful coordination |
| `firestore.rules` | Data access security for all 28 collections — requires careful review |
| `storage.rules` | Storage security — questionnaire files |
| `functions/src/middleware/auth.ts` | Auth and role validation — never relax without explicit review |
| `.env` | Local Firebase API keys — gitignored; rotate if leaked |
| `functions/.env.tpl` | 1Password secret references — safe to commit, but do not add actual secrets |

## Standard Compliance Notes

- **`tests/` placeholder:** Main test suites are at `./vitest.config.ts` (frontend) and `functions/` (backend). The root `tests/` directory is a standard placeholder per the AI Agent Tooling Standard.
- **`dist/` present:** Vite-generated build artifacts. The `check_dist_not_modified` CI script validates no direct edits.
- **`config/` and `mappings/` directories:** Non-standard top-level directories with documented purpose (see above). Justified as necessary for this project's configuration and data mapping needs.
- **`.github/` directory:** CI/CD pipeline configuration. Not in the standard layout but standard practice for GitHub Actions.
- **`.cursor/plans/`:** Cursor-internal plan tracking files (machine-generated). Not the same as the root `plans/` directory. The CI check for tool folder instructions excludes `.plan.md` files.
- **`scripts/ci/` present but CI pipeline may not be fully connected:** CI enforcement scripts exist per the standard. The `check-no-public-secrets.mjs` script runs as part of `npm test`.
